## Introduction

BlaC's **plugin system** provides a powerful way to extend and customize the framework's behavior without modifying core code. Plugins allow you to hook into the Bloc lifecycle, monitor state changes, add analytics, implement validation rules, measure performance, and much more.

Plugins are **non-intrusive** — they observe and react to Bloc lifecycle events without altering the core state management logic. This makes them perfect for cross-cutting concerns like logging, debugging, monitoring, and analytics.

### What You'll Learn

- How to create custom plugins using the `BlacPlugin` interface
- The difference between system-level and Bloc-level plugins
- Plugin lifecycle hooks: `onBlocCreated`, `onStateChanged`, `onBlocDisposed`
- Real-world plugin examples: analytics, performance monitoring, validation
- Best practices for plugin architecture and testing
- Common pitfalls and how to avoid them

## Understanding the Plugin System

BlaC plugins implement the `BlacPlugin` interface, which defines three core lifecycle hooks:

```typescript
interface BlacPlugin {
  name: string;
  version?: string;

  // Called when a new Bloc/Cubit instance is created
  onBlocCreated?(bloc: BlocBase<any>): void;

  // Called whenever a Bloc/Cubit emits a new state
  onStateChanged?(bloc: BlocBase<any>, previousState: any, currentState: any): void;

  // Called when a Bloc/Cubit is disposed
  onBlocDisposed?(bloc: BlocBase<any>): void;
}
```

<GuideArticleNote title="Lifecycle Hook Guarantees">
- `onBlocCreated` is called **once** when the Bloc is instantiated
- `onStateChanged` is called **every time** state changes (can be frequent!)
- `onBlocDisposed` is called **once** when the Bloc is cleaned up
- Hooks are called **synchronously** in the order plugins were registered
</GuideArticleNote>

### Plugin Types

BlaC supports two types of plugins:

**1. System-Level Plugins** — Applied globally to **all** Blocs:

```typescript
import { Blac } from '@blac/core';

const analyticsPlugin = new AnalyticsPlugin();
Blac.addPlugin(analyticsPlugin);
// Now tracks ALL Blocs in your application
```

**2. Bloc-Level Plugins** — Applied to **specific** Bloc classes:

```typescript
class UserCubit extends Cubit<UserState> {
  static plugins = [new UserValidationPlugin()];
  // Only applies to UserCubit instances
}
```

## Creating Your First Plugin

Let's start with a simple logging plugin that tracks all Bloc activity:

```typescript
import { BlacPlugin, BlocBase } from '@blac/core';

class LoggingPlugin implements BlacPlugin {
  name = 'LoggingPlugin';
  version = '1.0.0';

  onBlocCreated(bloc: BlocBase<any>) {
    console.log(`[${bloc._name}] Created with initial state:`, bloc.state);
  }

  onStateChanged(bloc: BlocBase<any>, previousState: any, currentState: any) {
    console.log(`[${bloc._name}] State changed:`);
    console.log('  Previous:', previousState);
    console.log('  Current:', currentState);
  }

  onBlocDisposed(bloc: BlocBase<any>) {
    console.log(`[${bloc._name}] Disposed`);
  }
}

// Register globally
Blac.addPlugin(new LoggingPlugin());
```

Now **every** Bloc in your application will log its lifecycle events to the console!

## Real-World Plugin Examples

### Example 1: Analytics Plugin

Track user interactions and state changes for analytics:

```typescript
class AnalyticsPlugin implements BlacPlugin {
  name = 'AnalyticsPlugin';

  private events: Array<{
    event: string;
    blocName: string;
    timestamp: number;
    metadata?: any;
  }> = [];

  onBlocCreated(bloc: BlocBase<any>) {
    this.recordEvent('bloc_created', bloc);

    // Send to analytics service
    this.sendToAnalytics({
      event: 'bloc_lifecycle',
      action: 'created',
      blocType: bloc.constructor.name,
    });
  }

  onStateChanged(bloc: BlocBase<any>, previousState: any, currentState: any) {
    this.recordEvent('state_changed', bloc, {
      previous: this.sanitize(previousState),
      current: this.sanitize(currentState),
    });

    // Track specific business metrics
    if (bloc._name.includes('Purchase')) {
      this.trackPurchaseMetrics(currentState);
    }
  }

  onBlocDisposed(bloc: BlocBase<any>) {
    this.recordEvent('bloc_disposed', bloc);

    // Calculate session metrics
    const lifetime = this.calculateLifetime(bloc);
    this.sendToAnalytics({
      event: 'bloc_session',
      blocType: bloc.constructor.name,
      lifetime,
    });
  }

  private recordEvent(event: string, bloc: BlocBase<any>, metadata?: any) {
    this.events.push({
      event,
      blocName: bloc._name,
      timestamp: Date.now(),
      metadata,
    });
  }

  private sanitize(state: any): any {
    // Remove sensitive data before logging
    const { password, creditCard, ...safe } = state;
    return safe;
  }

  private sendToAnalytics(data: any) {
    // Send to your analytics service (e.g., Mixpanel, Amplitude)
    // analytics.track(data);
  }

  private trackPurchaseMetrics(state: any) {
    if (state.status === 'completed') {
      this.sendToAnalytics({
        event: 'purchase_completed',
        amount: state.total,
        items: state.items.length,
      });
    }
  }

  private calculateLifetime(bloc: BlocBase<any>): number {
    const created = this.events.find(
      e => e.event === 'bloc_created' && e.blocName === bloc._name
    );
    return created ? Date.now() - created.timestamp : 0;
  }

  // Export events for debugging
  getEvents() {
    return this.events;
  }

  clearEvents() {
    this.events = [];
  }
}
```

**Usage:**

```typescript
const analyticsPlugin = new AnalyticsPlugin();
Blac.addPlugin(analyticsPlugin);

// Later, export analytics data
const events = analyticsPlugin.getEvents();
console.log(`Tracked ${events.length} lifecycle events`);
```

### Example 2: Performance Monitoring Plugin

Measure how long state transitions take:

```typescript
class PerformancePlugin implements BlacPlugin {
  name = 'PerformancePlugin';

  private measurements = new Map<string, {
    stateChangeCount: number;
    totalTime: number;
    slowestChange: number;
  }>();

  private lastStateChangeTime = new Map<string, number>();

  onBlocCreated(bloc: BlocBase<any>) {
    this.measurements.set(bloc._name, {
      stateChangeCount: 0,
      totalTime: 0,
      slowestChange: 0,
    });
    this.lastStateChangeTime.set(bloc._name, Date.now());
  }

  onStateChanged(bloc: BlocBase<any>, previousState: any, currentState: any) {
    const now = Date.now();
    const lastTime = this.lastStateChangeTime.get(bloc._name) || now;
    const duration = now - lastTime;

    const stats = this.measurements.get(bloc._name);
    if (stats) {
      stats.stateChangeCount++;
      stats.totalTime += duration;
      stats.slowestChange = Math.max(stats.slowestChange, duration);
    }

    this.lastStateChangeTime.set(bloc._name, now);

    // Warn if state change took too long
    if (duration > 100) {
      console.warn(
        `[Performance] Slow state change in ${bloc._name}: ${duration}ms`
      );
    }
  }

  onBlocDisposed(bloc: BlocBase<any>) {
    const stats = this.measurements.get(bloc._name);
    if (stats && stats.stateChangeCount > 0) {
      const avgTime = stats.totalTime / stats.stateChangeCount;
      console.log(`[Performance] ${bloc._name} Lifetime Stats:`);
      console.log(`  Total state changes: ${stats.stateChangeCount}`);
      console.log(`  Average duration: ${avgTime.toFixed(2)}ms`);
      console.log(`  Slowest change: ${stats.slowestChange}ms`);
    }

    this.measurements.delete(bloc._name);
    this.lastStateChangeTime.delete(bloc._name);
  }

  // Get performance report
  getReport() {
    const report: Record<string, any> = {};
    this.measurements.forEach((stats, blocName) => {
      report[blocName] = {
        changes: stats.stateChangeCount,
        avgTime: stats.totalTime / Math.max(1, stats.stateChangeCount),
        slowest: stats.slowestChange,
      };
    });
    return report;
  }
}
```

**Usage:**

```typescript
const perfPlugin = new PerformancePlugin();
Blac.addPlugin(perfPlugin);

// Later, get performance report
const report = perfPlugin.getReport();
console.table(report);
```

### Example 3: Validation Plugin

Add runtime validation for state transitions:

```typescript
type ValidationRule<T> = (state: T) => string | null;

class ValidationPlugin implements BlacPlugin {
  name = 'ValidationPlugin';

  private validators = new Map<string, ValidationRule<any>>();
  private violations: Array<{
    blocName: string;
    violation: string;
    state: any;
    timestamp: number;
  }> = [];

  // Register validation rules for specific Blocs
  registerValidator<T>(blocClassName: string, validator: ValidationRule<T>) {
    this.validators.set(blocClassName, validator);
  }

  onStateChanged(bloc: BlocBase<any>, previousState: any, currentState: any) {
    const className = bloc.constructor.name;
    const validator = this.validators.get(className);

    if (validator) {
      const error = validator(currentState);
      if (error) {
        this.violations.push({
          blocName: bloc._name,
          violation: error,
          state: currentState,
          timestamp: Date.now(),
        });

        console.error(
          `[Validation] ${bloc._name} violated constraint: ${error}`
        );
        console.error('  Invalid state:', currentState);

        // Optionally throw in development
        if (process.env.NODE_ENV === 'development') {
          throw new Error(`State validation failed: ${error}`);
        }
      }
    }
  }

  getViolations() {
    return this.violations;
  }

  clearViolations() {
    this.violations = [];
  }
}
```

**Usage:**

```typescript
const validationPlugin = new ValidationPlugin();

// Register validation rules
validationPlugin.registerValidator('UserCubit', (state) => {
  if (!state.email.includes('@')) {
    return 'Email must be valid';
  }
  if (state.age < 0 || state.age > 120) {
    return 'Age must be between 0 and 120';
  }
  return null; // Valid
});

validationPlugin.registerValidator('CartCubit', (state) => {
  if (state.items.length > 100) {
    return 'Cart cannot exceed 100 items';
  }
  if (state.total < 0) {
    return 'Total cannot be negative';
  }
  return null;
});

Blac.addPlugin(validationPlugin);

// Later, check for violations
const violations = validationPlugin.getViolations();
if (violations.length > 0) {
  console.error('Found validation violations:', violations);
}
```

### Example 4: State History Plugin

Track state history for debugging and time-travel:

```typescript
class StateHistoryPlugin implements BlacPlugin {
  name = 'StateHistoryPlugin';

  private history = new Map<string, Array<{
    state: any;
    timestamp: number;
  }>>();

  private maxHistorySize = 50; // Limit memory usage

  onBlocCreated(bloc: BlocBase<any>) {
    this.history.set(bloc._name, [{
      state: structuredClone(bloc.state),
      timestamp: Date.now(),
    }]);
  }

  onStateChanged(bloc: BlocBase<any>, previousState: any, currentState: any) {
    const blocHistory = this.history.get(bloc._name) || [];

    // Add new state to history
    blocHistory.push({
      state: structuredClone(currentState),
      timestamp: Date.now(),
    });

    // Limit history size
    if (blocHistory.length > this.maxHistorySize) {
      blocHistory.shift(); // Remove oldest entry
    }

    this.history.set(bloc._name, blocHistory);
  }

  onBlocDisposed(bloc: BlocBase<any>) {
    // Optionally keep history even after disposal for debugging
    // Or clear it: this.history.delete(bloc._name);
  }

  // Get state history for a specific Bloc
  getHistory(blocName: string) {
    return this.history.get(blocName) || [];
  }

  // Get state at a specific time
  getStateAt(blocName: string, timestamp: number) {
    const blocHistory = this.history.get(blocName);
    if (!blocHistory) return null;

    // Find closest state before or at timestamp
    return blocHistory
      .filter(entry => entry.timestamp <= timestamp)
      .pop()?.state || null;
  }

  // Export all history for debugging
  exportHistory() {
    const exported: Record<string, any> = {};
    this.history.forEach((history, blocName) => {
      exported[blocName] = history;
    });
    return exported;
  }

  clearHistory() {
    this.history.clear();
  }
}
```

**Usage:**

```typescript
const historyPlugin = new StateHistoryPlugin();
Blac.addPlugin(historyPlugin);

// Later, inspect history
const userHistory = historyPlugin.getHistory('UserCubit_1');
console.log(`User went through ${userHistory.length} state changes`);

// Time-travel debugging
const stateAt = historyPlugin.getStateAt('UserCubit_1', Date.now() - 5000);
console.log('State 5 seconds ago:', stateAt);
```

## Bloc-Specific Plugins

For plugins that only apply to specific Blocs, use the static `plugins` property:

```typescript
class UserValidationPlugin implements BlacPlugin {
  name = 'UserValidationPlugin';

  onStateChanged(bloc: BlocBase<any>, previousState: any, currentState: any) {
    // This only runs for UserCubit instances
    if (!currentState.email.includes('@')) {
      console.error('Invalid email format');
    }
  }
}

class UserCubit extends Cubit<UserState> {
  static plugins = [new UserValidationPlugin()];

  constructor() {
    super({ email: '', name: '' });
  }

  updateEmail = (email: string) => {
    this.emit({ ...this.state, email });
    // UserValidationPlugin will validate automatically
  };
}
```

This approach keeps validation logic **separate** from business logic while still being **specific** to the Bloc that needs it.

## Best Practices

<GuideArticleNote title="Do">
- ✅ Keep plugins **focused** on a single concern (logging, analytics, etc.)
- ✅ Use system-level plugins for cross-cutting concerns
- ✅ Use Bloc-level plugins for domain-specific logic
- ✅ **Sanitize sensitive data** before logging or sending to analytics
- ✅ Implement **cleanup logic** in `onBlocDisposed` to prevent memory leaks
- ✅ Make plugins **configurable** (enable/disable, log levels, etc.)
- ✅ Use `structuredClone()` when storing state to avoid mutation issues
- ✅ Test plugins independently from your Blocs
</GuideArticleNote>

<GuideArticleNote title="Don't">
- ❌ Modify Bloc state from plugins (plugins should **observe**, not **mutate**)
- ❌ Perform expensive operations in `onStateChanged` (it's called frequently!)
- ❌ Store unlimited history (use circular buffers or size limits)
- ❌ Log sensitive data (passwords, tokens, credit cards)
- ❌ Throw errors from plugins (catch and log instead)
- ❌ Create tight coupling between plugins and specific Bloc implementations
- ❌ Forget to clean up resources in `onBlocDisposed`
</GuideArticleNote>

## Performance Considerations

Plugin hooks run **synchronously** on every state change. For high-frequency Blocs, this can impact performance:

### ⚠️ Performance Anti-Pattern

```typescript
class BadPlugin implements BlacPlugin {
  name = 'BadPlugin';

  onStateChanged(bloc: BlocBase<any>, previousState: any, currentState: any) {
    // ❌ BAD: Expensive operation on every state change
    const diff = deepDiff(previousState, currentState);
    fs.writeFileSync(`/logs/${bloc._name}.json`, JSON.stringify(diff));

    // ❌ BAD: Synchronous network request
    fetch('/api/analytics', {
      method: 'POST',
      body: JSON.stringify(currentState),
    });
  }
}
```

### ✅ Performance Best Practice

```typescript
class GoodPlugin implements BlacPlugin {
  name = 'GoodPlugin';

  private queue: any[] = [];
  private flushInterval: NodeJS.Timeout;

  constructor() {
    // Batch and flush periodically
    this.flushInterval = setInterval(() => this.flush(), 5000);
  }

  onStateChanged(bloc: BlocBase<any>, previousState: any, currentState: any) {
    // ✅ GOOD: Quick operation, batched for later
    this.queue.push({
      blocName: bloc._name,
      timestamp: Date.now(),
      state: currentState,
    });

    // Limit queue size
    if (this.queue.length > 1000) {
      this.queue.shift();
    }
  }

  private async flush() {
    if (this.queue.length === 0) return;

    const batch = this.queue.splice(0, 100); // Process in batches

    try {
      await fetch('/api/analytics/batch', {
        method: 'POST',
        body: JSON.stringify(batch),
      });
    } catch (error) {
      console.error('Failed to send analytics:', error);
    }
  }

  cleanup() {
    clearInterval(this.flushInterval);
    this.flush(); // Final flush
  }
}
```

## Testing Plugins

Plugins should be tested independently:

```typescript
import { describe, it, expect, vi } from 'vitest';
import { Cubit } from '@blac/core';

describe('AnalyticsPlugin', () => {
  it('tracks bloc creation', () => {
    const plugin = new AnalyticsPlugin();
    const cubit = new Cubit(0);

    plugin.onBlocCreated(cubit);

    const events = plugin.getEvents();
    expect(events).toHaveLength(1);
    expect(events[0].event).toBe('bloc_created');
  });

  it('tracks state changes', () => {
    const plugin = new AnalyticsPlugin();
    const cubit = new Cubit({ count: 0 });

    plugin.onBlocCreated(cubit);
    plugin.onStateChanged(cubit, { count: 0 }, { count: 1 });
    plugin.onStateChanged(cubit, { count: 1 }, { count: 2 });

    const events = plugin.getEvents();
    expect(events).toHaveLength(3); // created + 2 changes
    expect(events.filter(e => e.event === 'state_changed')).toHaveLength(2);
  });

  it('sanitizes sensitive data', () => {
    const plugin = new AnalyticsPlugin();
    const cubit = new Cubit({
      username: 'john',
      password: 'secret123',
    });

    plugin.onStateChanged(cubit, {}, cubit.state);

    const events = plugin.getEvents();
    expect(events[0].metadata.current).not.toHaveProperty('password');
    expect(events[0].metadata.current).toHaveProperty('username');
  });

  it('cleans up on disposal', () => {
    const plugin = new PerformancePlugin();
    const cubit = new Cubit(0);

    plugin.onBlocCreated(cubit);
    plugin.onBlocDisposed(cubit);

    const report = plugin.getReport();
    expect(report[cubit._name]).toBeUndefined();
  });
});
```

## Advanced: Conditional Plugin Registration

You can conditionally register plugins based on environment or feature flags:

```typescript
import { Blac } from '@blac/core';

// Development-only plugins
if (process.env.NODE_ENV === 'development') {
  Blac.addPlugin(new LoggingPlugin());
  Blac.addPlugin(new StateHistoryPlugin());
}

// Production plugins
if (process.env.NODE_ENV === 'production') {
  Blac.addPlugin(new AnalyticsPlugin({
    apiKey: process.env.ANALYTICS_API_KEY,
  }));
  Blac.addPlugin(new PerformancePlugin({
    sampleRate: 0.1, // Only sample 10% of events
  }));
}

// Feature-flagged plugins
if (featureFlags.enableValidation) {
  const validationPlugin = new ValidationPlugin();
  // Register validators...
  Blac.addPlugin(validationPlugin);
}
```

## Key Takeaways

- **Non-Intrusive Extension**: Plugins observe Bloc lifecycle without modifying core logic
- **Three Lifecycle Hooks**: `onBlocCreated`, `onStateChanged`, `onBlocDisposed`
- **Two Plugin Types**: System-level (global) and Bloc-level (specific)
- **Performance Matters**: Batch expensive operations, avoid synchronous work in hooks
- **Separation of Concerns**: Keep cross-cutting concerns (logging, analytics) in plugins
- **Testable**: Plugins can be tested independently from your Blocs
- **Configurable**: Make plugins environment-aware and feature-flaggable
- **Memory Management**: Clean up resources in `onBlocDisposed`
