# Advanced Form Validation

Real-world forms need more than simple "field is filled" checks. You need **async validation** (checking username availability), **cross-field validation** (passwords match), **password strength indicators**, and **clear user feedback**. Let's build production-ready validation with BlaC.

## What You'll Learn

- Field-level synchronous validation patterns
- Async validation with debouncing (username availability checks)
- Cross-field validation (password confirmation)
- Password strength indicators
- Touch state for smart error display
- Validation summary and form-level status

## Three Types of Validation

Complex forms require three distinct validation strategies:

### 1. Synchronous Field Validation

**Instant, local validation rules** that check format, length, or patterns:

```typescript
class FormCubit extends Cubit<FormState> {
  // Boolean check: Is field valid?
  get isUsernameValid(): boolean {
    const { username } = this.state;
    return (
      username.length >= 3 &&
      username.length <= 20 &&
      /^[a-zA-Z0-9_]+$/.test(username)
    );
  }

  // Error messages: What's wrong?
  get usernameErrors(): string[] {
    const { username } = this.state;
    const errors: string[] = [];

    if (username.length > 0 && username.length < 3) {
      errors.push('Username must be at least 3 characters');
    }
    if (username.length > 20) {
      errors.push('Username must be 20 characters or less');
    }
    if (username.length > 0 && !/^[a-zA-Z0-9_]+$/.test(username)) {
      errors.push('Only letters, numbers, and underscores allowed');
    }

    return errors;
  }
}
```

**Pattern**: Separate `isXValid` (boolean) from `xErrors` (string array) for maximum flexibility.

### 2. Asynchronous Validation

**Server-side checks** that require API calls (username availability, email verification):

```typescript
interface FormState {
  username: string;
  usernameValidating: boolean; // Show loading spinner
  usernameAvailable: boolean | null; // null = not checked yet
}

class FormCubit extends Cubit<FormState> {
  private usernameCheckTimeout: NodeJS.Timeout | null = null;

  setUsername = (username: string) => {
    this.patch({ username, usernameAvailable: null });

    // Trigger async validation after user stops typing
    if (username.length >= 3) {
      this.validateUsernameAsync(username);
    }
  };

  validateUsernameAsync = async (username: string) => {
    // Clear previous timeout (debouncing)
    if (this.usernameCheckTimeout) {
      clearTimeout(this.usernameCheckTimeout);
    }

    // Wait 500ms before checking (avoid excessive API calls)
    this.usernameCheckTimeout = setTimeout(async () => {
      if (!this.isUsernameValid) return; // Only check if format is valid

      this.patch({ usernameValidating: true });

      try {
        // API call to check availability
        const response = await fetch(`/api/check-username?username=${username}`);
        const { available } = await response.json();

        this.patch({
          usernameValidating: false,
          usernameAvailable: available,
        });
      } catch (error) {
        this.patch({
          usernameValidating: false,
          usernameAvailable: null, // Reset on error
        });
      }
    }, 500); // Debounce delay
  };

  onDispose = () => {
    // Clean up timeout to prevent memory leaks
    if (this.usernameCheckTimeout) {
      clearTimeout(this.usernameCheckTimeout);
      this.usernameCheckTimeout = null;
    }
  };
}
```

<GuideArticleNote title="Debouncing is Critical">
Without debouncing, **every keystroke triggers an API call**. For "administrator" (13 characters), that's 13 API calls!

**With debouncing**: Wait 500ms after the user stops typing. One API call total.
</GuideArticleNote>

### 3. Cross-Field Validation

**Validation that depends on multiple fields**:

```typescript
class FormCubit extends Cubit<FormState> {
  // Password confirmation
  get isConfirmPasswordValid(): boolean {
    const { password, confirmPassword } = this.state;
    return confirmPassword.length > 0 && password === confirmPassword;
  }

  get confirmPasswordErrors(): string[] {
    const { password, confirmPassword } = this.state;
    if (confirmPassword.length > 0 && password !== confirmPassword) {
      return ['Passwords do not match'];
    }
    return [];
  }

  // Date range validation
  get isDateRangeValid(): boolean {
    const { startDate, endDate } = this.state;
    if (!startDate || !endDate) return false;
    return new Date(endDate) >= new Date(startDate);
  }

  // Discount validation
  get isDiscountValid(): boolean {
    const { discountAmount, originalPrice } = this.state;
    return discountAmount >= 0 && discountAmount <= originalPrice;
  }
}
```

**Pattern**: Cross-field getters access multiple state properties to check consistency.

## Password Strength Indicator

Password strength is a computed property based on multiple criteria:

```typescript
class FormCubit extends Cubit<FormState> {
  get passwordStrength(): 'weak' | 'medium' | 'strong' | null {
    const { password } = this.state;
    if (password.length === 0) return null;

    let strength = 0;

    // Length checks
    if (password.length >= 8) strength++;
    if (password.length >= 12) strength++;

    // Character variety checks
    if (/[a-z]/.test(password) && /[A-Z]/.test(password)) strength++; // Mixed case
    if (/[0-9]/.test(password)) strength++; // Numbers
    if (/[^a-zA-Z0-9]/.test(password)) strength++; // Special chars

    // Calculate final strength
    if (strength <= 2) return 'weak';
    if (strength <= 3) return 'medium';
    return 'strong';
  }

  get passwordErrors(): string[] {
    const { password } = this.state;
    const errors: string[] = [];

    if (password.length > 0 && password.length < 8) {
      errors.push('Password must be at least 8 characters');
    }
    if (password.length > 0 && !/[a-z]/.test(password)) {
      errors.push('Password must contain lowercase letters');
    }
    if (password.length > 0 && !/[A-Z]/.test(password)) {
      errors.push('Password must contain uppercase letters');
    }
    if (password.length > 0 && !/[0-9]/.test(password)) {
      errors.push('Password should contain numbers');
    }
    if (password.length > 0 && !/[^a-zA-Z0-9]/.test(password)) {
      errors.push('Password should contain special characters');
    }

    return errors;
  }
}
```

**Visual feedback in React**:

```tsx
function PasswordStrengthIndicator({ strength }: { strength: 'weak' | 'medium' | 'strong' | null }) {
  if (!strength) return null;

  const config = {
    weak: { color: 'bg-red-500', width: 'w-1/3', label: 'Weak' },
    medium: { color: 'bg-yellow-500', width: 'w-2/3', label: 'Medium' },
    strong: { color: 'bg-green-500', width: 'w-full', label: 'Strong' },
  };

  const { color, width, label } = config[strength];

  return (
    <div>
      <div>
        <span>Password Strength:</span>
        <span>{label}</span>
      </div>
      <div>
        <div className={`h-full ${color} ${width} transition-all duration-300`} />
      </div>
    </div>
  );
}
```

<GuideArticleNote title="Immediate vs Delayed Feedback">
Password strength is shown **immediately** as the user types (aggressive validation), while password errors are shown **onBlur** (optimistic validation).

This combination provides helpful guidance without being frustrating.
</GuideArticleNote>

## Touch State Pattern

Only show errors **after the user has interacted with a field**:

```tsx
function RegistrationForm() {
  const [state, cubit] = useBloc(FormCubit);

  // Track which fields the user has interacted with
  const [touched, setTouched] = useState({
    username: false,
    email: false,
    password: false,
  });

  const markTouched = (field: keyof typeof touched) => {
    setTouched((prev) => ({ ...prev, [field]: true }));
  };

  const handleSubmit = async (e: React.FormEvent) => {
    e.preventDefault();

    // Mark ALL fields as touched on submit
    setTouched({
      username: true,
      email: true,
      password: true,
    });

    await cubit.submit();
  };

  return (
    <form onSubmit={handleSubmit}>
      <input
        value={state.username}
        onChange={(e) => cubit.setUsername(e.target.value)}
        onBlur={() => markTouched('username')} // Mark touched on blur
        className={touched.username && cubit.usernameErrors.length > 0 ? 'error' : ''}
      />
      {/* Only show errors if field has been touched */}
      {touched.username && cubit.usernameErrors.map((error, idx) => (
        <p key={idx}>{error}</p>
      ))}
    </form>
  );
}
```

**Why this works**:
- User doesn't see errors until they've interacted with the field
- Errors appear onBlur (when they move to the next field)
- All errors are shown on submit attempt

<GuideArticleNote title="Flat State for Touch Tracking">
You might be tempted to store touch state in the Cubit:

```typescript
// ❌ BAD: Nested state
interface FormState {
  fields: {
    username: { value: string; touched: boolean };
  };
}

// ✅ GOOD: Flat state with separate tracking
interface FormState {
  username: string;
  usernameTouched: boolean;
}
```

Or, even better, **track touch state in React** (as shown above) since it's UI-specific, not business logic.
</GuideArticleNote>

## Form-Level Validation

Combine all field validations into overall form status:

```typescript
class FormCubit extends Cubit<FormState> {
  get isFormValid(): boolean {
    return (
      this.isUsernameValid &&
      this.state.usernameAvailable === true && // Async validation passed
      this.isEmailValid &&
      this.isPasswordValid &&
      this.isConfirmPasswordValid &&
      this.isAgeValid &&
      this.state.acceptTerms // Terms checkbox checked
    );
  }

  // Use this to disable the submit button
  get canSubmit(): boolean {
    return this.isFormValid && !this.state.isSubmitting;
  }

  submit = async () => {
    // Guard: Don't submit if form is invalid
    if (!this.isFormValid) {
      this.patch({
        submitError: 'Please fix all validation errors before submitting',
      });
      return;
    }

    this.patch({ isSubmitting: true, submitError: null });

    try {
      const response = await fetch('/api/register', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
          username: this.state.username,
          email: this.state.email,
          password: this.state.password,
        }),
      });

      if (!response.ok) {
        throw new Error('Registration failed');
      }

      this.patch({
        isSubmitting: false,
        submitSuccess: true,
      });
    } catch (error) {
      this.patch({
        isSubmitting: false,
        submitError: error.message,
      });
    }
  };
}
```

**Usage in React**:

```tsx
<button
  type="submit"
  disabled={!cubit.canSubmit}
>
  {state.isSubmitting ? 'Submitting...' : 'Submit'}
</button>

{state.submitError && (
  <div>{state.submitError}</div>
)}

{state.submitSuccess && (
  <div>Registration successful!</div>
)}
```

## Real-World Example: Email Verification

Async validation with retry and timeout:

```typescript
class EmailVerificationCubit extends Cubit<EmailVerificationState> {
  private controller: AbortController | null = null;

  verifyEmail = async (email: string) => {
    // Cancel previous request
    if (this.controller) {
      this.controller.abort();
    }

    this.controller = new AbortController();
    this.patch({ verifying: true, error: null });

    try {
      const response = await fetch('/api/verify-email', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ email }),
        signal: this.controller.signal, // Allow cancellation
      });

      const { exists, deliverable } = await response.json();

      this.patch({
        verifying: false,
        emailExists: exists,
        emailDeliverable: deliverable,
      });
    } catch (error) {
      if (error.name !== 'AbortError') {
        this.patch({
          verifying: false,
          error: 'Failed to verify email',
        });
      }
    }
  };

  onDispose = () => {
    // Clean up
    if (this.controller) {
      this.controller.abort();
      this.controller = null;
    }
  };
}
```

## Testing Validation Logic

Test Cubit validation in isolation:

```typescript
import { describe, test, expect } from 'vitest';

describe('FormValidationCubit', () => {
  test('username validation enforces length', () => {
    const cubit = new FormValidationCubit();

    cubit.setUsername('ab'); // Too short
    expect(cubit.isUsernameValid).toBe(false);
    expect(cubit.usernameErrors).toContain('Username must be at least 3 characters');

    cubit.setUsername('abc'); // Just right
    expect(cubit.isUsernameValid).toBe(true);
    expect(cubit.usernameErrors).toHaveLength(0);

    cubit.dispose();
  });

  test('password strength calculation', () => {
    const cubit = new FormValidationCubit();

    cubit.setPassword('pass'); // Weak
    expect(cubit.passwordStrength).toBe('weak');

    cubit.setPassword('Password1'); // Medium
    expect(cubit.passwordStrength).toBe('medium');

    cubit.setPassword('P@ssw0rd!'); // Strong
    expect(cubit.passwordStrength).toBe('strong');

    cubit.dispose();
  });

  test('cross-field password confirmation', () => {
    const cubit = new FormValidationCubit();

    cubit.setPassword('Password123!');
    cubit.setConfirmPassword('Password123'); // Doesn't match

    expect(cubit.isConfirmPasswordValid).toBe(false);
    expect(cubit.confirmPasswordErrors).toContain('Passwords do not match');

    cubit.setConfirmPassword('Password123!'); // Matches
    expect(cubit.isConfirmPasswordValid).toBe(true);
    expect(cubit.confirmPasswordErrors).toHaveLength(0);

    cubit.dispose();
  });

  test('form is invalid until all fields pass', () => {
    const cubit = new FormValidationCubit();

    expect(cubit.isFormValid).toBe(false);

    // Fill out all fields correctly
    cubit.setUsername('validuser');
    cubit.patch({ usernameAvailable: true }); // Simulate async check
    cubit.setEmail('user@example.com');
    cubit.setPassword('Password123!');
    cubit.setConfirmPassword('Password123!');
    cubit.setAge('25');
    cubit.setAcceptTerms(true);

    expect(cubit.isFormValid).toBe(true);

    cubit.dispose();
  });
});
```

## Common Pitfalls

### Pitfall 1: Not Debouncing Async Validation

```typescript
// ❌ BAD: API call on every keystroke
setUsername = (username: string) => {
  this.patch({ username });
  this.checkUsernameAvailability(username); // Too many calls!
};

// ✅ GOOD: Debounced API calls
setUsername = (username: string) => {
  this.patch({ username });

  if (this.usernameCheckTimeout) {
    clearTimeout(this.usernameCheckTimeout);
  }

  this.usernameCheckTimeout = setTimeout(() => {
    this.checkUsernameAvailability(username);
  }, 500);
};
```

### Pitfall 2: Storing Validation Results in State

```typescript
// ❌ BAD: Validation state can become stale
interface FormState {
  email: string;
  emailValid: boolean; // Can get out of sync with email!
}

// ✅ GOOD: Derive validation from current state
interface FormState {
  email: string;
}

class FormCubit extends Cubit<FormState> {
  get emailValid(): boolean {
    return /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(this.state.email);
  }
}
```

### Pitfall 3: Showing All Errors Immediately

```typescript
// ❌ BAD: Errors show before user has interacted
{cubit.emailErrors.map(error => <p>{error}</p>)}

// ✅ GOOD: Only show errors after blur or submit
{touched.email && cubit.emailErrors.map(error => <p>{error}</p>)}
```

### Pitfall 4: Forgetting Cleanup

```typescript
// ❌ BAD: Memory leak from timeout
class FormCubit extends Cubit<FormState> {
  private timeout: NodeJS.Timeout | null = null;

  validateAsync = () => {
    this.timeout = setTimeout(/* ... */, 500);
  };
  // No onDispose! Timeout never cleared.
}

// ✅ GOOD: Clean up resources
class FormCubit extends Cubit<FormState> {
  private timeout: NodeJS.Timeout | null = null;

  onDispose = () => {
    if (this.timeout) {
      clearTimeout(this.timeout);
      this.timeout = null;
    }
  };
}
```

## Best Practices

1. **Separate boolean checks from error arrays**: `isEmailValid` (boolean) + `emailErrors` (string[])
2. **Debounce async validation**: Wait 300-500ms after user stops typing
3. **Show loading indicators**: Track `usernameValidating` to show spinners
4. **Use nullable results**: `usernameAvailable: boolean | null` distinguishes "not checked" from "checked and unavailable"
5. **Touch state in React**: Track which fields user has interacted with
6. **Only show errors after interaction**: onBlur or onSubmit, not onChange
7. **Clean up resources**: Clear timeouts and abort controllers in `onDispose`
8. **Guard submit**: Check `if (!this.isFormValid) return;` at the start
9. **Provide specific errors**: "Password must contain uppercase letters" not "Invalid password"
10. **Test validation in isolation**: Test Cubit logic without rendering components

<GuideArticleNote title="Security: Client-Side Validation is for UX Only">
**Always validate on the server too.** Client-side validation provides immediate feedback to users, but malicious users can bypass it.

Never trust client-side validation for security. Always validate and sanitize data on the backend.
</GuideArticleNote>

## Key Takeaways

- **Three validation types**: Synchronous (instant), asynchronous (API), cross-field (multiple dependencies)
- **Getters for validation**: Derive validation state from field values, don't store it
- **Debounce async checks**: Wait for user to stop typing (500ms delay)
- **Touch state pattern**: Only show errors after user interaction (onBlur/onSubmit)
- **Password strength**: Computed property based on multiple criteria
- **Form-level status**: Combine all validations into `isFormValid`
- **Clean up timers**: Always implement `onDispose` for async operations
