# Filtering & Sorting

## Introduction

Search, filter, and sort functionality are **table stakes** for any data-heavy application. Users expect to find what they need quickly—whether it's products in an e-commerce store, emails in their inbox, or rows in a data table.

The challenge? Implementing filters that **work together seamlessly**, perform well with large datasets, and provide an **instant, responsive experience**.

In this guide, you'll learn professional patterns for building powerful search and filter systems that scale.

### What You'll Learn

- Implement multi-field text search across your data
- Combine multiple filters with logical AND/OR operations
- Build dynamic sorting with ascending/descending order
- Use computed properties to derive filtered results efficiently
- Optimize performance for large datasets (1000+ items)
- Debounce search inputs for better UX and performance

## The Core Pattern: Filter Criteria + Computed Results

The key insight: **Store filter criteria in state, compute filtered results on-demand.**

```typescript
interface ProductState {
  products: Product[]; // Original data

  // Filter criteria (stored in state)
  searchQuery: string;
  categoryFilter: string;
  priceRange: { min: number; max: number };
  inStockOnly: boolean;

  // Sorting criteria
  sortBy: 'name' | 'price' | 'rating';
  sortOrder: 'asc' | 'desc';

  // ❌ Don't store this:
  // filteredProducts: Product[];  // Duplication!
}

class ProductFilterCubit extends Cubit<ProductState> {
  // ✅ Compute filtered results on-demand
  get filteredAndSortedProducts(): Product[] {
    let results = [...this.state.products];

    // Apply all filters
    results = this.applyFilters(results);

    // Apply sorting
    results = this.applySorting(results);

    return results;
  }
}
```

**Why this works:**
- **Single source of truth**: Original data lives in `products`
- **No duplication**: Filtered results are computed, not stored
- **Always in sync**: Changes to criteria instantly reflect in results
- **Composable**: Easy to add/remove filters without breaking state

## Multi-Field Text Search

Text search should check **multiple fields** for matches:

```typescript
interface Product {
  id: number;
  name: string;
  category: string;
  description: string;
  tags: string[];
}

class ProductFilterCubit extends Cubit<ProductState> {
  setSearchQuery = (query: string) => {
    this.patch({ searchQuery: query.toLowerCase() });
  };

  get filteredProducts(): Product[] {
    let results = [...this.state.products];

    // Multi-field search
    if (this.state.searchQuery) {
      const query = this.state.searchQuery;

      results = results.filter((product) => {
        // Check name
        if (product.name.toLowerCase().includes(query)) return true;

        // Check category
        if (product.category.toLowerCase().includes(query)) return true;

        // Check description
        if (product.description.toLowerCase().includes(query)) return true;

        // Check tags
        if (product.tags.some(tag => tag.toLowerCase().includes(query))) return true;

        return false;
      });
    }

    return results;
  }
}
```

<GuideArticleNote title="Case-Insensitive Search">
Always normalize search queries to lowercase using `.toLowerCase()` for case-insensitive matching. Store the normalized query in state to avoid repeated `.toLowerCase()` calls during filtering.
</GuideArticleNote>

## Combining Multiple Filters

Filters typically work with **logical AND** (all filters must match):

```typescript
get filteredProducts(): Product[] {
  let results = [...this.state.products];

  // Filter 1: Search query (multi-field)
  if (this.state.searchQuery) {
    const query = this.state.searchQuery;
    results = results.filter((p) =>
      p.name.toLowerCase().includes(query) ||
      p.category.toLowerCase().includes(query)
    );
  }

  // Filter 2: Category (exact match)
  if (this.state.categoryFilter !== 'all') {
    results = results.filter((p) => p.category === this.state.categoryFilter);
  }

  // Filter 3: Price range (numeric range)
  results = results.filter(
    (p) => p.price >= this.state.priceRange.min &&
           p.price <= this.state.priceRange.max
  );

  // Filter 4: Stock status (boolean)
  if (this.state.inStockOnly) {
    results = results.filter((p) => p.inStock === true);
  }

  return results;
}
```

Each filter **narrows down** the results from the previous filter. The order doesn't matter for correctness, but you can optimize by putting the most selective filters first.

## Dynamic Sorting

Sorting requires **two pieces of state**: what to sort by, and the direction (ascending/descending).

```typescript
interface ProductState {
  sortBy: 'name' | 'price' | 'rating';
  sortOrder: 'asc' | 'desc';
}

class ProductFilterCubit extends Cubit<ProductState> {
  setSortBy = (sortBy: 'name' | 'price' | 'rating') => {
    this.patch({ sortBy });
  };

  toggleSortOrder = () => {
    this.patch({
      sortOrder: this.state.sortOrder === 'asc' ? 'desc' : 'asc'
    });
  };

  get sortedProducts(): Product[] {
    const results = [...this.state.products];

    results.sort((a, b) => {
      let comparison = 0;

      // Determine comparison based on sort field
      switch (this.state.sortBy) {
        case 'name':
          // String comparison (alphabetical)
          comparison = a.name.localeCompare(b.name);
          break;

        case 'price':
          // Numeric comparison
          comparison = a.price - b.price;
          break;

        case 'rating':
          // Numeric comparison
          comparison = a.rating - b.rating;
          break;
      }

      // Apply sort direction
      return this.state.sortOrder === 'asc' ? comparison : -comparison;
    });

    return results;
  }
}
```

<GuideArticleNote title="String Sorting with localeCompare">
For **string sorting**, use `.localeCompare()` instead of `<` or `>`:

```typescript
// ❌ WRONG: Doesn't handle accents, diacritics, or locale-specific rules
comparison = a.name < b.name ? -1 : 1;

// ✅ CORRECT: Handles international characters properly
comparison = a.name.localeCompare(b.name);
```

`.localeCompare()` respects language-specific sorting rules (e.g., "ä" sorts correctly in German).
</GuideArticleNote>

## Performance Optimization

### 1. Debounce Search Input

Don't filter on **every keystroke**—wait until the user stops typing:

```tsx
function ProductSearch() {
  const [state, cubit] = useBloc(ProductFilterCubit);
  const [inputValue, setInputValue] = useState('');

  // Debounce the search query
  useEffect(() => {
    const timer = setTimeout(() => {
      cubit.setSearchQuery(inputValue);
    }, 300); // Wait 300ms after user stops typing

    return () => clearTimeout(timer); // Cleanup on unmount or value change
  }, [inputValue]);

  return (
    <input
      value={inputValue}
      onChange={(e) => setInputValue(e.target.value)}
      placeholder="Search products..."
    />
  );
}
```

**Benefits:**
- **Reduces re-renders**: Only filters when user pauses
- **Better UX**: No lag while typing
- **Lower CPU usage**: Fewer filter operations

### 2. Memoize Expensive Filters

For **complex filters** or **large datasets** (10,000+ items), consider memoization:

```typescript
import { useMemo } from 'react';

function ProductList() {
  const [state, cubit] = useBloc(ProductFilterCubit);

  // Memoize filtered results (only recompute when dependencies change)
  const filteredProducts = useMemo(
    () => cubit.filteredAndSortedProducts,
    [state.searchQuery, state.categoryFilter, state.priceRange, state.sortBy, state.sortOrder]
  );

  return (
    <div>
      {filteredProducts.map(product => (
        <ProductCard key={product.id} product={product} />
      ))}
    </div>
  );
}
```

**Note**: This is only needed for **very large lists**. For most use cases, computed properties (getters) are fast enough.

### 3. Virtual Scrolling for Large Lists

### 4. Web Workers for Heavy Filtering

For **extremely large datasets** or **complex filters**, offload work to a Web Worker:

```typescript
// filter-worker.ts
self.onmessage = (e) => {
  const { products, filters } = e.data;

  // Perform filtering in background thread
  const filtered = products.filter((p: Product) => {
    // Apply all filters
    return matchesFilters(p, filters);
  });

  self.postMessage(filtered);
};

// In your Cubit
class ProductFilterCubit extends Cubit<ProductState> {
  private worker = new Worker(new URL('./filter-worker.ts', import.meta.url));

  filterAsync = () => {
    this.patch({ loading: true });

    this.worker.postMessage({
      products: this.state.products,
      filters: {
        searchQuery: this.state.searchQuery,
        categoryFilter: this.state.categoryFilter,
        // ... other filters
      },
    });

    this.worker.onmessage = (e) => {
      this.patch({
        filteredProducts: e.data,
        loading: false,
      });
    };
  };
}
```

## Real-World Example: E-Commerce Product Catalog

Here's a complete, production-ready implementation:

```typescript
interface Product {
  id: string;
  name: string;
  category: string;
  brand: string;
  price: number;
  rating: number;
  reviewCount: number;
  inStock: boolean;
  tags: string[];
  createdAt: Date;
}

interface CatalogState {
  products: Product[];
  searchQuery: string;
  categories: string[];
  brands: string[];
  priceRange: { min: number; max: number };
  minRating: number;
  inStockOnly: boolean;
  sortBy: 'relevance' | 'price' | 'rating' | 'newest';
  sortOrder: 'asc' | 'desc';
}

class CatalogCubit extends Cubit<CatalogState> {
  constructor(products: Product[]) {
    super({
      products,
      searchQuery: '',
      categories: [],
      brands: [],
      priceRange: { min: 0, max: Infinity },
      minRating: 0,
      inStockOnly: false,
      sortBy: 'relevance',
      sortOrder: 'desc',
    });
  }

  // Filter methods
  setSearchQuery = (query: string) => {
    this.patch({ searchQuery: query.toLowerCase() });
  };

  toggleCategory = (category: string) => {
    const categories = this.state.categories.includes(category)
      ? this.state.categories.filter(c => c !== category)
      : [...this.state.categories, category];
    this.patch({ categories });
  };

  setPriceRange = (min: number, max: number) => {
    this.patch({ priceRange: { min, max } });
  };

  setMinRating = (rating: number) => {
    this.patch({ minRating: rating });
  };

  toggleInStockOnly = () => {
    this.patch({ inStockOnly: !this.state.inStockOnly });
  };

  // Sort methods
  setSortBy = (sortBy: CatalogState['sortBy']) => {
    this.patch({ sortBy });
  };

  toggleSortOrder = () => {
    this.patch({
      sortOrder: this.state.sortOrder === 'asc' ? 'desc' : 'asc'
    });
  };

  // Reset all filters
  resetFilters = () => {
    this.patch({
      searchQuery: '',
      categories: [],
      brands: [],
      priceRange: { min: 0, max: Infinity },
      minRating: 0,
      inStockOnly: false,
    });
  };

  // Main computed property: apply all filters and sorting
  get filteredProducts(): Product[] {
    let results = [...this.state.products];

    // Apply search filter (multi-field)
    if (this.state.searchQuery) {
      const query = this.state.searchQuery;
      results = results.filter((p) =>
        p.name.toLowerCase().includes(query) ||
        p.category.toLowerCase().includes(query) ||
        p.brand.toLowerCase().includes(query) ||
        p.tags.some(tag => tag.toLowerCase().includes(query))
      );
    }

    // Apply category filter (OR logic for multiple categories)
    if (this.state.categories.length > 0) {
      results = results.filter((p) =>
        this.state.categories.includes(p.category)
      );
    }

    // Apply brand filter (OR logic)
    if (this.state.brands.length > 0) {
      results = results.filter((p) =>
        this.state.brands.includes(p.brand)
      );
    }

    // Apply price range filter
    results = results.filter(
      (p) => p.price >= this.state.priceRange.min &&
             p.price <= this.state.priceRange.max
    );

    // Apply minimum rating filter
    if (this.state.minRating > 0) {
      results = results.filter((p) => p.rating >= this.state.minRating);
    }

    // Apply stock filter
    if (this.state.inStockOnly) {
      results = results.filter((p) => p.inStock);
    }

    // Apply sorting
    results = this.applySorting(results);

    return results;
  }

  private applySorting = (products: Product[]): Product[] => {
    const sorted = [...products];

    sorted.sort((a, b) => {
      let comparison = 0;

      switch (this.state.sortBy) {
        case 'relevance':
          // Sort by review count * rating (popularity score)
          const scoreA = a.reviewCount * a.rating;
          const scoreB = b.reviewCount * b.rating;
          comparison = scoreB - scoreA;
          break;

        case 'price':
          comparison = a.price - b.price;
          break;

        case 'rating':
          comparison = a.rating - b.rating;
          break;

        case 'newest':
          comparison = a.createdAt.getTime() - b.createdAt.getTime();
          break;
      }

      return this.state.sortOrder === 'asc' ? comparison : -comparison;
    });

    return sorted;
  };

  // Computed statistics
  get activeFilterCount(): number {
    let count = 0;
    if (this.state.searchQuery) count++;
    if (this.state.categories.length > 0) count++;
    if (this.state.brands.length > 0) count++;
    if (this.state.priceRange.min > 0 || this.state.priceRange.max < Infinity) count++;
    if (this.state.minRating > 0) count++;
    if (this.state.inStockOnly) count++;
    return count;
  }

  get availableCategories(): string[] {
    return Array.from(new Set(this.state.products.map(p => p.category))).sort();
  }

  get availableBrands(): string[] {
    return Array.from(new Set(this.state.products.map(p => p.brand))).sort();
  }
}
```

## Best Practices

### 1. Provide Clear Feedback

Show users **how many results** match their filters:

```tsx
<div>
  {cubit.filteredProducts.length} of {state.products.length} products
  {cubit.activeFilterCount > 0 && (
    <button onClick={cubit.resetFilters}>
      Clear {cubit.activeFilterCount} filters
    </button>
  )}
</div>
```

### 2. Persist Filters in URL

Allow users to **bookmark** or **share** filtered views:

```typescript
// Encode filters in URL query params
const searchParams = new URLSearchParams();
searchParams.set('search', state.searchQuery);
searchParams.set('category', state.categoryFilter);
searchParams.set('sort', state.sortBy);

history.pushState(null, '', `?${searchParams.toString()}`);

// Restore filters from URL on mount
useEffect(() => {
  const params = new URLSearchParams(window.location.search);
  const search = params.get('search');
  const category = params.get('category');
  const sort = params.get('sort');

  if (search) cubit.setSearchQuery(search);
  if (category) cubit.setCategoryFilter(category);
  if (sort) cubit.setSortBy(sort as any);
}, []);
```

### 3. Highlight Search Matches

Show users **why** a result matched their search:

```tsx
function highlightMatch(text: string, query: string): string {
  if (!query) return text;

  const regex = new RegExp(`(${query})`, 'gi');
  return text.replace(regex, '<mark>$1</mark>');
}

<h3 dangerouslySetInnerHTML={{
  __html: highlightMatch(product.name, state.searchQuery)
}} />
```

<GuideArticleNote title="Security Warning">
When using `dangerouslySetInnerHTML`, **always sanitize user input** to prevent XSS attacks. Use a library like `DOMPurify`:

```typescript
import DOMPurify from 'dompurify';

const sanitizedHTML = DOMPurify.sanitize(highlightMatch(product.name, state.searchQuery));
```
</GuideArticleNote>

## Key Takeaways

- **Store criteria, compute results**: Never duplicate filtered data in state
- **Use computed properties** (getters) for derived data
- **Combine filters with AND logic**: Each filter narrows results
- **Debounce search inputs** for better performance and UX
- **Sort with `.localeCompare()`** for proper internationalization
- **Virtual scrolling** for lists with 1,000+ visible items
- **Show active filter count** and provide easy reset
- **Persist filters in URL** for shareable, bookmarkable views
