## Introduction

By default, BlaC automatically disposes Cubit and Bloc instances when their last consumer component unmounts. This is efficient for memory management, but sometimes you need state to persist even when no components are actively using it. That's where the **KeepAlive pattern** comes in.

KeepAlive allows you to tell BlaC: "Keep this instance alive indefinitely, even with zero consumers." This is perfect for global app state, user sessions, cached data, and background services that should persist throughout your app's lifecycle.

### What You'll Learn

- How BlaC's default lifecycle management works
- When and why to use the KeepAlive pattern
- Implementing KeepAlive with `static keepAlive = true`
- Common use cases: auth, shopping carts, WebSocket connections
- Memory considerations and best practices
- Testing KeepAlive Cubits effectively
- When NOT to use KeepAlive

## Default Lifecycle Behavior

Before we dive into KeepAlive, let's understand the default behavior:

### Regular Cubits

```typescript
class RegularCounterCubit extends Cubit<number> {
  // No keepAlive property - default behavior

  constructor() {
    super(0);
    console.log('✅ RegularCounterCubit constructed');
  }

  dispose = async () => {
    console.log('🗑️  RegularCounterCubit disposed');
    return super.dispose();
  };
}
```

**Lifecycle:**
1. **First consumer mounts** → Instance is created
2. **Additional consumers mount** → They share the same instance
3. **Some consumers unmount** → Instance remains (still has consumers)
4. **Last consumer unmounts** → Instance is **disposed** and garbage collected

<GuideArticleNote title="Default is Efficient">
The default behavior is memory-efficient. Cubits are only kept alive when needed, automatically cleaned up when unused. For most use cases, this is exactly what you want.
</GuideArticleNote>

### The Problem

The default lifecycle doesn't work well for:
- **Global state** that should outlive component lifecycles
- **User sessions** that persist across route changes
- **Shopping carts** that shouldn't reset when navigating away
- **Background services** like WebSocket connections

## The KeepAlive Solution

Add one line to your Cubit/Bloc class:

```typescript
class PersistentCounterCubit extends Cubit<number> {
  static keepAlive = true; // This is the magic line!

  constructor() {
    super(0);
    console.log('✅ PersistentCounterCubit constructed');
  }

  dispose = async () => {
    console.log('🗑️  This will NEVER be called automatically!');
    return super.dispose();
  };
}
```

**KeepAlive Lifecycle:**
1. **First consumer mounts** → Instance is created
2. **Additional consumers mount** → They share the same instance
3. **All consumers unmount** → Instance **stays alive** (not disposed)
4. **New consumers mount** → They get the **same instance** with preserved state

<GuideArticleNote title="Key Difference">
With KeepAlive, the instance is **never disposed** unless you explicitly call `dispose()`. The state persists indefinitely, surviving all component unmounts.
</GuideArticleNote>

## Implementation Patterns

### 1. Global Authentication State

One of the most common use cases for KeepAlive:

```typescript
interface AuthState {
  isAuthenticated: boolean;
  user: User | null;
  token: string | null;
}

class AuthCubit extends Cubit<AuthState> {
  static keepAlive = true; // Persist across all routes

  constructor() {
    super({
      isAuthenticated: false,
      user: null,
      token: null,
    });

    // Restore auth from localStorage on construction
    this.restoreSession();
  }

  private restoreSession = async () => {
    const token = localStorage.getItem('auth_token');
    if (token) {
      try {
        const user = await this.validateToken(token);
        this.emit({ isAuthenticated: true, user, token });
      } catch (error) {
        // Token invalid, clear it
        localStorage.removeItem('auth_token');
      }
    }
  };

  login = async (credentials: Credentials) => {
    const { token, user } = await api.login(credentials);

    // Persist to localStorage
    localStorage.setItem('auth_token', token);

    // Update state
    this.emit({ isAuthenticated: true, user, token });
  };

  logout = () => {
    localStorage.removeItem('auth_token');
    this.emit({ isAuthenticated: false, user: null, token: null });
  };

  private validateToken = async (token: string): Promise<User> => {
    const response = await fetch('/api/validate', {
      headers: { Authorization: `Bearer ${token}` },
    });
    if (!response.ok) throw new Error('Invalid token');
    return response.json();
  };
}
```

**Usage in any component:**

```tsx
function Header() {
  const [state, auth] = useBloc(AuthCubit);

  return (
    <header>
      {state.isAuthenticated ? (
        <>
          <span>Welcome, {state.user?.name}</span>
          <button onClick={auth.logout}>Logout</button>
        </>
      ) : (
        <button onClick={() => navigate('/login')}>Login</button>
      )}
    </header>
  );
}

function LoginPage() {
  const [state, auth] = useBloc(AuthCubit);
  // Same AuthCubit instance as Header!

  const handleLogin = () => {
    auth.login({ username, password });
  };

  return <LoginForm onSubmit={handleLogin} />;
}
```

<GuideArticleNote title="Why KeepAlive for Auth?">
Without KeepAlive, your auth state would be lost when navigating between pages if no components currently use it. With KeepAlive, auth persists throughout the app lifecycle, providing a seamless experience.
</GuideArticleNote>

### 2. Shopping Cart (Session State)

Carts should persist even when users navigate away:

```typescript
interface CartItem {
  id: string;
  name: string;
  price: number;
  quantity: number;
}

interface CartState {
  items: CartItem[];
  total: number;
}

class ShoppingCartCubit extends Cubit<CartState> {
  static keepAlive = true; // Persist cart across navigation

  constructor() {
    super({ items: [], total: 0 });
    this.restoreCart();
  }

  private restoreCart = () => {
    const saved = localStorage.getItem('shopping_cart');
    if (saved) {
      const cart = JSON.parse(saved);
      this.emit(cart);
    }
  };

  private persist = (state: CartState) => {
    localStorage.setItem('shopping_cart', JSON.stringify(state));
  };

  addItem = (item: CartItem) => {
    // Check if item already exists
    const existingIndex = this.state.items.findIndex((i) => i.id === item.id);

    let newItems: CartItem[];
    if (existingIndex >= 0) {
      // Update quantity
      newItems = this.state.items.map((i, idx) =>
        idx === existingIndex
          ? { ...i, quantity: i.quantity + item.quantity }
          : i
      );
    } else {
      // Add new item
      newItems = [...this.state.items, item];
    }

    const total = this.calculateTotal(newItems);
    const newState = { items: newItems, total };

    this.emit(newState);
    this.persist(newState);
  };

  removeItem = (itemId: string) => {
    const newItems = this.state.items.filter((i) => i.id !== itemId);
    const total = this.calculateTotal(newItems);
    const newState = { items: newItems, total };

    this.emit(newState);
    this.persist(newState);
  };

  updateQuantity = (itemId: string, quantity: number) => {
    const newItems = this.state.items.map((i) =>
      i.id === itemId ? { ...i, quantity } : i
    );
    const total = this.calculateTotal(newItems);
    const newState = { items: newItems, total };

    this.emit(newState);
    this.persist(newState);
  };

  clearCart = () => {
    const newState = { items: [], total: 0 };
    this.emit(newState);
    localStorage.removeItem('shopping_cart');
  };

  private calculateTotal = (items: CartItem[]): number => {
    return items.reduce((sum, item) => sum + item.price * item.quantity, 0);
  };
}
```

**Usage:**

```tsx
// Product page - add to cart
function ProductPage() {
  const [cart, cartCubit] = useBloc(ShoppingCartCubit);

  const handleAddToCart = () => {
    cartCubit.addItem({
      id: product.id,
      name: product.name,
      price: product.price,
      quantity: 1,
    });
  };

  return <button onClick={handleAddToCart}>Add to Cart ({cart.items.length})</button>;
}

// Cart page - view and modify cart
function CartPage() {
  const [cart, cartCubit] = useBloc(ShoppingCartCubit);
  // Same instance as ProductPage!

  return (
    <div>
      {cart.items.map((item) => (
        <CartItem
          key={item.id}
          item={item}
          onRemove={() => cartCubit.removeItem(item.id)}
          onUpdateQuantity={(qty) => cartCubit.updateQuantity(item.id, qty)}
        />
      ))}
      <div>Total: ${cart.total.toFixed(2)}</div>
    </div>
  );
}
```

### 3. WebSocket Connection (Background Service)

Keep WebSocket connections alive across component lifecycles:

```typescript
interface WebSocketState {
  connected: boolean;
  messages: Message[];
  error: string | null;
}

class WebSocketCubit extends Cubit<WebSocketState> {
  static keepAlive = true; // Keep connection alive
  private ws: WebSocket | null = null;
  private reconnectTimer: NodeJS.Timeout | null = null;

  constructor() {
    super({
      connected: false,
      messages: [],
      error: null,
    });

    // Auto-connect on construction
    this.connect();
  }

  connect = () => {
    if (this.ws?.readyState === WebSocket.OPEN) return;

    this.ws = new WebSocket('wss://api.example.com/ws');

    this.ws.onopen = () => {
      console.log('🔌 WebSocket connected');
      this.patch({ connected: true, error: null });
      this.clearReconnectTimer();
    };

    this.ws.onmessage = (event) => {
      const message = JSON.parse(event.data);
      this.patch({
        messages: [...this.state.messages, message],
      });
    };

    this.ws.onerror = (error) => {
      console.error('WebSocket error:', error);
      this.patch({ error: 'Connection error' });
    };

    this.ws.onclose = () => {
      console.log('🔌 WebSocket closed, reconnecting...');
      this.patch({ connected: false });
      this.scheduleReconnect();
    };
  };

  private scheduleReconnect = () => {
    this.clearReconnectTimer();
    this.reconnectTimer = setTimeout(() => {
      console.log('🔄 Attempting reconnect...');
      this.connect();
    }, 5000);
  };

  private clearReconnectTimer = () => {
    if (this.reconnectTimer) {
      clearTimeout(this.reconnectTimer);
      this.reconnectTimer = null;
    }
  };

  send = (data: any) => {
    if (this.ws?.readyState === WebSocket.OPEN) {
      this.ws.send(JSON.stringify(data));
    } else {
      console.warn('Cannot send: WebSocket not connected');
    }
  };

  clearMessages = () => {
    this.patch({ messages: [] });
  };

  // Cleanup when explicitly disposed
  onDispose = async () => {
    console.log('🗑️  Cleaning up WebSocket connection');
    this.clearReconnectTimer();
    this.ws?.close();
    this.ws = null;
    return super.onDispose();
  };
}
```

**Usage:**

```tsx
// Chat component
function Chat() {
  const [state, ws] = useBloc(WebSocketCubit);

  const handleSend = (message: string) => {
    ws.send({ type: 'chat', text: message });
  };

  return (
    <div>
      <div>Status: {state.connected ? '🟢 Connected' : '🔴 Disconnected'}</div>
      {state.messages.map((msg, i) => (
        <div key={i}>{msg.text}</div>
      ))}
      <input onSubmit={handleSend} />
    </div>
  );
}
```

<GuideArticleNote title="Cleanup is Important">
Even with KeepAlive, always implement `onDispose()` for cleanup. While the Cubit won't be automatically disposed, you may manually dispose it (e.g., on app shutdown), and cleanup logic ensures resources like WebSockets are properly closed.
</GuideArticleNote>

## Memory Considerations

KeepAlive Cubits **stay in memory forever** (until explicitly disposed), so use them wisely:

### Memory Impact

```typescript
class KeepAliveCubit extends Cubit<State> {
  static keepAlive = true;

  constructor() {
    super(initialState);
    // This instance will NEVER be garbage collected automatically
    // It stays in memory even with zero consumers
  }
}
```

**Good use of KeepAlive** (small, essential state):
- User authentication (a few KB)
- App settings/configuration
- Shopping cart (tens of items max)
- Feature flags

**Bad use of KeepAlive** (large, temporary data):
- ❌ Paginated list of 10,000 items
- ❌ Cached API responses for every endpoint
- ❌ Temporary form state
- ❌ Modal/dialog state

<GuideArticleNote title="Rule of Thumb">
Only use KeepAlive for state that:
1. **Must** persist across component lifecycles
2. Is **small** in size (< 1MB ideally)
3. Is **truly global** (used throughout the app)
4. Has a **lifetime matching the app lifecycle**

For everything else, use regular Cubits.
</GuideArticleNote>

## Manual Disposal

Even KeepAlive Cubits can be manually disposed when needed:

```typescript
class CacheCubit extends Cubit<CacheState> {
  static keepAlive = true;

  constructor() {
    super({ items: [] });
  }

  clearAndDispose = async () => {
    // Clear all state
    this.emit({ items: [] });

    // Manually dispose the instance
    await this.dispose();
  };
}
```

**When to manually dispose:**
- User logs out → dispose auth state
- Cache invalidation → dispose cached data
- App shutdown → cleanup all KeepAlive instances
- Testing → reset state between tests

## Testing KeepAlive Cubits

Testing KeepAlive Cubits requires special care because instances persist between tests:

### Problem: State Leaks Between Tests

```typescript
// ❌ BAD: Tests will interfere with each other
describe('AuthCubit', () => {
  it('should log in', () => {
    const auth = new AuthCubit();
    auth.login({ username: 'test', password: 'pass' });

    expect(auth.state.isAuthenticated).toBe(true);
    // KeepAlive means instance persists!
  });

  it('should log out', () => {
    const auth = new AuthCubit();
    // This might get the SAME instance from previous test!
    // State could still show authenticated=true

    auth.logout();
    expect(auth.state.isAuthenticated).toBe(false);
  });
});
```

### Solution: Reset Between Tests

```typescript
// ✅ GOOD: Explicitly dispose between tests
describe('AuthCubit', () => {
  let auth: AuthCubit;

  beforeEach(() => {
    auth = new AuthCubit();
  });

  afterEach(async () => {
    // Manually dispose to clean up
    await auth.dispose();
  });

  it('should log in', () => {
    auth.login({ username: 'test', password: 'pass' });
    expect(auth.state.isAuthenticated).toBe(true);
  });

  it('should log out', () => {
    // Fresh instance thanks to afterEach cleanup
    auth.logout();
    expect(auth.state.isAuthenticated).toBe(false);
  });
});
```

**Alternative: Use regular Cubits in tests**

```typescript
// Create a test-only version without KeepAlive
class TestAuthCubit extends AuthCubit {
  static keepAlive = false; // Override for testing
}

describe('AuthCubit', () => {
  it('should log in', () => {
    const auth = new TestAuthCubit();
    // Automatically disposed after test
    auth.login({ username: 'test', password: 'pass' });
    expect(auth.state.isAuthenticated).toBe(true);
  });
});
```

## Best Practices

### ✅ Do Use KeepAlive For

1. **User Authentication**
   ```typescript
   class AuthCubit extends Cubit<AuthState> {
     static keepAlive = true; // ✅ Essential global state
   }
   ```

2. **App Configuration**
   ```typescript
   class ConfigCubit extends Cubit<ConfigState> {
     static keepAlive = true; // ✅ Feature flags, theme, settings
   }
   ```

3. **Shopping Cart/Draft Data**
   ```typescript
   class CartCubit extends Cubit<CartState> {
     static keepAlive = true; // ✅ Persist across navigation
   }
   ```

4. **Background Services**
   ```typescript
   class WebSocketCubit extends Cubit<WSState> {
     static keepAlive = true; // ✅ Maintain connection
   }
   ```

### ❌ Don't Use KeepAlive For

1. **Modal/Dialog State**
   ```typescript
   class ModalCubit extends Cubit<ModalState> {
     // ❌ NO keepAlive - dispose when modal closes
   }
   ```

2. **Component-Local State**
   ```typescript
   class DropdownCubit extends Cubit<boolean> {
     // ❌ NO keepAlive - tied to component
   }
   ```

3. **Temporary Filters/Sorting**
   ```typescript
   class FilterCubit extends Cubit<FilterState> {
     // ❌ NO keepAlive - page-specific
   }
   ```

4. **Paginated Lists**
   ```typescript
   class ProductListCubit extends Cubit<ProductListState> {
     // ❌ NO keepAlive - large data, page-specific
   }
   ```

## Common Pitfalls

### 1. Overusing KeepAlive

```typescript
// ❌ BAD: Every Cubit has KeepAlive
class FormCubit extends Cubit<FormState> {
  static keepAlive = true; // ❌ Not needed for form state!
}

class ModalCubit extends Cubit<ModalState> {
  static keepAlive = true; // ❌ Modals should reset!
}

class ListFilterCubit extends Cubit<FilterState> {
  static keepAlive = true; // ❌ Filters are page-specific!
}
```

**Impact:** Memory bloat, stale state, unexpected behavior

### 2. Forgetting to Implement onDispose

```typescript
// ❌ BAD: No cleanup
class WebSocketCubit extends Cubit<WSState> {
  static keepAlive = true;
  private ws: WebSocket;

  constructor() {
    super({ connected: false });
    this.ws = new WebSocket('wss://api.example.com');
    // WebSocket will never be closed!
  }
}

// ✅ GOOD: Cleanup on dispose
class WebSocketCubit extends Cubit<WSState> {
  static keepAlive = true;
  private ws: WebSocket;

  constructor() {
    super({ connected: false });
    this.ws = new WebSocket('wss://api.example.com');
  }

  onDispose = async () => {
    this.ws?.close(); // Cleanup resource
    return super.onDispose();
  };
}
```

### 3. Storing Large Data

```typescript
// ❌ BAD: Large data in KeepAlive
class CacheCubit extends Cubit<CacheState> {
  static keepAlive = true;

  // Storing megabytes of data forever!
  constructor() {
    super({
      users: [], // 10,000 users
      products: [], // 50,000 products
      orders: [], // 100,000 orders
    });
  }
}
```

**Solution:** Use regular Cubits with intelligent caching, or cache in localStorage/IndexedDB.

## Key Takeaways

- **KeepAlive Pattern**: Add `static keepAlive = true` to persist instances indefinitely
- **Use Cases**: Authentication, settings, shopping carts, background services
- **Memory**: Use sparingly—only for small, essential global state
- **Lifecycle**: Instances never dispose automatically, survive all unmounts
- **Cleanup**: Always implement `onDispose()` for resource cleanup
- **Testing**: Manually dispose or use non-KeepAlive versions in tests
- **Best Practice**: Default to regular Cubits, only use KeepAlive when necessary
