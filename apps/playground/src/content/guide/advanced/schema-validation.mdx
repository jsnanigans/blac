# Schema Validation

Add **runtime type safety** to your Blocs and Cubits using schema validation libraries. BlaC supports any library that implements the [Standard Schema](https://standardschema.dev/) specification, including Zod, Valibot, and ArkType.

## What You'll Learn

- How to add schema validation to your state
- Validating external data (APIs, forms, localStorage)
- Using helper methods for safe validation
- When to use schemas vs TypeScript types

## Why Schema Validation?

TypeScript provides **compile-time** type safety, but can't validate data at **runtime**:

```typescript
// ❌ TypeScript can't prevent this at runtime
const apiData: User = await fetch('/api/user').then((r) => r.json());
// What if the API returns { name: 123, email: "invalid" }?

// ✅ Schema validation catches invalid data
const validatedData = UserSchema.parse(apiData);
// Throws if data is invalid!
```

**Use schema validation when:**

- Validating data from APIs, localStorage, or user input
- Enforcing business rules (age > 13, price > 0)
- Preventing data corruption from external sources

**Skip schema validation when:**

- State is simple and internal (counter, toggle)
- Data is always derived from trusted sources
- Performance is critical (schemas add overhead)

## Basic Usage

Add a `static schema` property to your Cubit or Bloc:

```typescript
import { z } from 'zod';
import { Cubit } from '@blac/core';

// 1. Define your schema
const UserSchema = z.object({
  name: z.string().min(1),
  age: z.number().int().min(0).max(150),
  email: z.string().email(),
});

type User = z.infer<typeof UserSchema>;

// 2. Add schema to Cubit
class UserCubit extends Cubit<User> {
  static schema = UserSchema; // ← That's it!

  constructor() {
    super({
      name: 'Alice',
      age: 30,
      email: 'alice@example.com',
    }); // ✅ Validated on construction
  }

  updateAge = (age: number) => {
    this.emit({ ...this.state, age }); // Internal update - no validation
  };

  // For external data, use emitValidated
  loadFromAPI = async (userId: string) => {
    const data = await fetch(`/api/users/${userId}`).then(r => r.json());
    this.emitValidated(data); // ✅ Validates external data
  };
}
```

**How it works:**

1. Define a schema using Zod, Valibot, ArkType, or any [Standard Schema](https://standardschema.dev/) library
2. Add it as `static schema` to your Cubit/Bloc
3. Initial state is validated in the constructor
4. Use `emitValidated()` to validate external data at boundaries
5. Invalid state throws `BlocValidationError` and state remains unchanged

**Important:** Regular `emit()` and `patch()` do **not** validate automatically. Use `emitValidated()` and `patchValidated()` when you need validation (typically for external data).

## Real-World Example: RPG Character Stats

Here's a simplified example showing how schema validation protects your state:

```typescript
import { z } from 'zod';
import { Cubit } from '@blac/core';

// Define schema with constraints
const CharacterSchema = z.object({
  health: z.number().int().min(0).max(1000),
  mana: z.number().int().min(0).max(500),
  level: z.number().int().min(1).max(100),
});

type CharacterStats = z.infer<typeof CharacterSchema>;

class CharacterCubit extends Cubit<CharacterStats> {
  static schema = CharacterSchema;

  constructor() {
    super({ health: 100, mana: 50, level: 1 }); // ✅ Validated on construction
  }

  // Internal game logic - no validation (you control these values)
  takeDamage = (amount: number) => {
    const newHealth = Math.max(0, this.state.health - amount); // Clamp to min
    this.emit({ ...this.state, health: newHealth });
  };

  heal = (amount: number) => {
    const newHealth = Math.min(1000, this.state.health + amount); // Clamp to max
    this.emit({ ...this.state, health: newHealth });
  };

  // External data from save file - validate to catch corrupted data
  loadFromSaveFile = (saveData: unknown) => {
    try {
      this.emitValidated(CharacterSchema.parse(saveData)); // ✅ Validates
      console.log('Save file loaded successfully');
    } catch (error) {
      console.error('Corrupted save file:', error);
      // Keep current state or reset to defaults
    }
  };

  // External data from multiplayer server
  syncFromServer = (serverData: unknown) => {
    this.emitValidated(CharacterSchema.parse(serverData)); // ✅ Validates
  };
}

// Usage
const character = new CharacterCubit();

// ✅ Internal logic with boundary checks
character.takeDamage(20);
console.log(character.state.health); // 80

character.takeDamage(1000); // Won't go negative (clamped to 0)
console.log(character.state.health); // 0

character.heal(10000); // Won't exceed max (clamped to 1000)
console.log(character.state.health); // 1000
```

**Key concepts:**

1. Schema defines constraints (min/max values, types)
2. Constructor validates initial state (boundaries)
3. `emitValidated()` validates external data (save files, server sync)
4. Regular `emit()` for internal logic (you control the values)
5. Invalid data throws `BlocValidationError`

**When to validate:**
- ✅ **External data** → Use `emitValidated()` (save files, API, user input)
- ✅ **Initial state** → Automatically validated in constructor
- ❌ **Internal logic** → Use regular `emit()` with your own bounds checking

## Actual Real-World examples:

Schema validation is essential whenever **external data** enters your application. Here are the most common scenarios:

### 1. API Response Validation

Protect your app from unexpected API changes or malformed responses:

```typescript
const UserSchema = z.object({
  id: z.string().uuid(),
  name: z.string().min(1),
  email: z.string().email(),
  role: z.enum(['admin', 'user', 'guest']),
});

type User = z.infer<typeof UserSchema>;

class UserProfileCubit extends Cubit<User | null> {
  static schema = z.nullable(UserSchema);

  loadUser = async (userId: string) => {
    const response = await fetch(`/api/users/${userId}`);
    const data = await response.json();

    try {
      // Validate API response - catches breaking API changes
      const validUser = UserSchema.parse(data);
      this.emit(validUser);
    } catch (error) {
      console.error('Invalid user data from API:', error);
      this.emit(null); // Graceful fallback
    }
  };
}
```

**Why it helps:** APIs change, return unexpected nulls, or have bugs. Schema validation catches these issues before they corrupt your state.

### 2. Form Input Validation

Enforce business rules and data constraints on user input:

```typescript
// Schema for validating form input
const RegistrationInputSchema = z.object({
  username: z
    .string()
    .min(3)
    .max(20)
    .regex(/^[a-zA-Z0-9_]+$/),
  email: z.string().email(),
  age: z.number().int().min(13, 'Must be 13 or older'),
  password: z.string().min(8),
  agreedToTerms: z.literal(true, {
    errorMap: () => ({ message: 'You must agree to terms' }),
  }),
});

// Schema for form state (includes errors and status)
const RegistrationStateSchema = z.object({
  isSubmitting: z.boolean(),
  errors: z.record(z.array(z.string())).optional(),
  success: z.boolean(),
});

type RegistrationState = z.infer<typeof RegistrationStateSchema>;

class RegistrationFormCubit extends Cubit<RegistrationState> {
  static schema = RegistrationStateSchema;

  constructor() {
    super({ isSubmitting: false, success: false });
  }

  submitForm = (formData: unknown) => {
    const result = RegistrationInputSchema.safeParse(formData);

    if (!result.success) {
      // Show user-friendly validation errors
      this.emit({
        ...this.state,
        errors: result.error.flatten().fieldErrors,
        isSubmitting: false,
      });
      return;
    }

    // Safe to proceed with validated data
    this.registerUser(result.data);
  };

  registerUser = async (data: z.infer<typeof RegistrationInputSchema>) => {
    this.emit({ ...this.state, isSubmitting: true });
    // ... register user logic
    this.emit({ isSubmitting: false, success: true });
  };
}
```

**Why it helps:** Enforces business rules (age restrictions, username format) that TypeScript alone can't express. Provides clear error messages for users.

### 3. localStorage/Session Storage Persistence

Validate persisted data to handle schema migrations and prevent stale data from breaking your app:

```typescript
const CartItemSchema = z.object({
  id: z.string(),
  name: z.string(),
  price: z.number().positive(),
  quantity: z.number().int().min(1),
});

const CartStateSchema = z.object({
  items: z.array(CartItemSchema),
  total: z.number(),
  // Add new fields with defaults for backward compatibility
  currency: z.string().default('USD'),
  lastUpdated: z.string().datetime().optional(),
});

type CartState = z.infer<typeof CartStateSchema>;

const DEFAULT_CART: CartState = {
  items: [],
  total: 0,
  currency: 'USD',
};

class ShoppingCartCubit extends Cubit<CartState> {
  static schema = CartStateSchema;

  constructor() {
    const savedCart = localStorage.getItem('cart');

    try {
      // Validate localStorage data - handles old schema versions
      const parsed = savedCart ? JSON.parse(savedCart) : null;
      const validated = parsed ? CartStateSchema.parse(parsed) : DEFAULT_CART;
      super(validated);
    } catch (error) {
      // Old/corrupted data - start fresh
      console.warn('Invalid cart data, resetting:', error);
      super(DEFAULT_CART);
      localStorage.removeItem('cart');
    }
  }

  // Auto-save to localStorage after state changes
  protected override emit(state: CartState): void {
    super.emit(state);
    localStorage.setItem('cart', JSON.stringify(state));
  }
}
```

**Why it helps:** Users might have old versions of your state schema stored. Validation prevents app crashes and enables graceful schema migrations with `.default()` and `.optional()`.

## When to Use emitValidated vs emit

Understanding when to validate is crucial for both performance and correctness.

### Use `emitValidated()` - Validate External Data

Use `emitValidated()` when data comes from **outside your application code**:

```typescript
class UserCubit extends Cubit<User> {
  static schema = UserSchema;

  // ✅ API responses - could be malformed
  loadFromAPI = async () => {
    const data = await fetch('/api/user').then(r => r.json());
    this.emitValidated(data);
  };

  // ✅ localStorage - could be old/corrupted
  loadFromStorage = () => {
    const data = JSON.parse(localStorage.getItem('user') || '{}');
    this.emitValidated(data);
  };

  // ✅ User input - untrusted
  submitForm = (formData: unknown) => {
    this.emitValidated(formData);
  };

  // ✅ WebSocket/server data
  onServerUpdate = (data: unknown) => {
    this.emitValidated(data);
  };
}
```

### Use `emit()` - Internal Transitions

Use regular `emit()` for **internal state changes** you control:

```typescript
class CounterCubit extends Cubit<number> {
  static schema = z.number().int().min(0).max(100);

  constructor() {
    super(0); // ✅ Validated in constructor
  }

  // ✅ Internal logic - you ensure correctness
  increment = () => {
    if (this.state < 100) {
      this.emit(this.state + 1); // No validation needed
    }
  };

  decrement = () => {
    if (this.state > 0) {
      this.emit(this.state - 1); // No validation needed
    }
  };

  // ✅ Computed values - derived from valid state
  reset = () => {
    this.emit(0); // No validation needed
  };
}
```

### Use `patchValidated()` - Partial External Updates

Use `patchValidated()` when patching with external data:

```typescript
class ProfileCubit extends Cubit<UserProfile> {
  static schema = UserProfileSchema;

  // ✅ Form submission - validate merged result
  updateProfile = (formData: Partial<UserProfile>) => {
    this.patchValidated(formData);
  };

  // ❌ Internal update - no validation needed
  incrementLoginCount = () => {
    this.patch({ loginCount: this.state.loginCount + 1 });
  };
}
```

### Rule of Thumb

**Ask:** "Did this data come from outside my code?"

- **Yes** (API, user input, storage) → `emitValidated()` or `patchValidated()`
- **No** (computed, derived, internal) → `emit()` or `patch()`

## Helper Methods

BlaC provides convenient methods for validation:

### `validate()` - Get Validation Result

```typescript
const cubit = new UserCubit();

const result = cubit.validate({
  name: 'Bob',
  age: 25,
  email: 'bob@example.com',
});

if (result.success) {
  console.log('Valid!', result.value);
} else {
  console.log('Invalid!', result.issues);
}
```

Returns `{ success: true, value: T }` or `{ success: false, issues: Issue[] }`

### `isValid()` - Boolean Check

```typescript
if (cubit.isValid(someData)) {
  cubit.emit(someData); // Safe to use
}
```

Returns `true` or `false`

### `parse()` - Validate and Throw

```typescript
try {
  const validated = cubit.parse(untrustedData);
  cubit.emit(validated);
} catch (error) {
  // Handle BlocValidationError
}
```

Throws `BlocValidationError` if invalid

### `safeParse()` - Non-Throwing Parse

```typescript
const result = cubit.safeParse(untrustedData);

if (result.success) {
  cubit.emit(result.value);
} else {
  console.error(result.error.issues);
}
```

Returns `{ success: true, value: T }` or `{ success: false, error: BlocValidationError }`

## Using Different Libraries

BlaC works with **any** library that implements [Standard Schema V1](https://standardschema.dev/). The three most popular are:

### Zod

```typescript
import { z } from 'zod';

const UserSchema = z.object({
  name: z.string(),
  age: z.number().int().min(0),
});

type User = z.infer<typeof UserSchema>;

class UserCubit extends Cubit<User> {
  static schema = UserSchema;
}
```

### Valibot

```typescript
import * as v from 'valibot';

const UserSchema = v.object({
  name: v.string(),
  age: v.pipe(v.number(), v.integer(), v.minValue(0)),
});

type User = v.InferOutput<typeof UserSchema>;

class UserCubit extends Cubit<User> {
  static schema = UserSchema;
}
```

### ArkType

```typescript
import { type } from 'arktype';

const UserSchema = type({
  name: 'string',
  age: 'number>=0',
});

type User = typeof UserSchema.infer;

class UserCubit extends Cubit<User> {
  static schema = UserSchema;
}
```

All three work identically with BlaC thanks to the [Standard Schema](https://standardschema.dev/) specification!

## Error Handling Patterns

### Pattern: Validate External Data

```typescript
const UserSchema = z.object({
  id: z.string().uuid(),
  name: z.string().min(1),
  email: z.string().email(),
});

type User = z.infer<typeof UserSchema>;

class DataCubit extends Cubit<User[]> {
  static schema = z.array(UserSchema);

  constructor() {
    super([]);
  }

  loadFromAPI = async () => {
    const response = await fetch('/api/users');
    const rawData = await response.json();

    try {
      // Validate API response
      const validated = z.array(UserSchema).parse(rawData);
      this.emit(validated); // ✅ Safe
    } catch (error) {
      if (error instanceof z.ZodError) {
        console.error('Invalid API data:', error.errors);
        // Use fallback or show error
        this.emit([]);
      }
    }
  };
}
```

### Pattern: Validate with User Feedback

```typescript
const FormInputSchema = z.object({
  email: z.string().email(),
  message: z.string().min(10),
});

const FormStateSchema = z.object({
  isSubmitting: z.boolean(),
  error: z.string().optional(),
  success: z.boolean(),
});

type FormState = z.infer<typeof FormStateSchema>;

class FormCubit extends Cubit<FormState> {
  static schema = FormStateSchema;

  constructor() {
    super({ isSubmitting: false, success: false });
  }

  submit = async (formData: unknown) => {
    const result = FormInputSchema.safeParse(formData);

    if (!result.success) {
      // Show user-friendly error
      this.emit({
        ...this.state,
        error: result.error.errors[0]?.message || 'Invalid input',
        isSubmitting: false,
      });
      return;
    }

    // Submit validated data
    this.emit({ ...this.state, isSubmitting: true, error: undefined });
    await api.submit(result.data);
    this.emit({ isSubmitting: false, success: true });
  };
}
```

### ❌ Anti-Pattern: Validating Already-Valid State

```typescript
// ❌ BAD: this.state is already valid!
submit = () => {
  const validated = this.parse(this.state); // Pointless!
  api.submit(validated);
};

// ✅ GOOD: Validate EXTERNAL data
submit = async () => {
  const apiData = await fetch('/api/data').then((r) => r.json());
  const validated = DataSchema.parse(apiData); // Useful!
  this.emit(validated);
};
```

**Remember:** Cubit state is **always valid** (schemas guarantee this). Only validate data from **external sources**.

## Schema Transformations

Schemas can transform data during validation:

```typescript
const TimestampSchema = z.object({
  createdAt: z.string().transform((s) => new Date(s)), // String → Date
  price: z.union([z.string(), z.number()]).transform(Number), // String or Number → Number
  tags: z.string().transform((s) => s.split(',')), // String → Array
});

// Input:  { createdAt: "2024-01-01", price: "99.99", tags: "a,b,c" }
// Output: { createdAt: Date, price: 99.99, tags: ["a", "b", "c"] }
```

Useful for:

- Parsing dates from ISO strings
- Coercing types from form data
- Normalizing data formats

## Testing with Schemas

Schemas make testing more robust:

```typescript
import { describe, test, expect } from 'vitest';

describe('UserCubit', () => {
  test('rejects invalid age', () => {
    const cubit = new UserCubit();

    expect(() => {
      cubit.emit({ ...cubit.state, age: -5 });
    }).toThrow(BlocValidationError);

    // State unchanged
    expect(cubit.state.age).toBe(30);
  });

  test('accepts valid data', () => {
    const cubit = new UserCubit();

    cubit.emit({ ...cubit.state, age: 25 });
    expect(cubit.state.age).toBe(25);
  });
});
```

## Performance Considerations

Schema validation adds **runtime overhead**:

```typescript
// Typical validation times (rough estimates)
// - Simple object (3-5 fields): ~0.01ms
// - Complex object (20+ fields): ~0.1ms
// - Array of 100 items: ~1-5ms
```

**When overhead matters:**

- ❌ High-frequency updates (60fps animations, mouse tracking)
- ❌ Large arrays updated frequently
- ✅ User interactions (form submissions, clicks)
- ✅ API responses (infrequent, important to validate)
- ✅ Loading from storage (one-time on mount)

**Optimization:** Skip validation for internal state updates, use it at system boundaries.

## Key Takeaways

- **Runtime safety:** Schemas catch invalid data that TypeScript can't
- **Standard Schema:** Use Zod, Valibot, ArkType, or any [standardschema.dev](https://standardschema.dev/) library
- **Add `static schema`:** Enables validation for your Cubit/Bloc
- **Constructor validates:** Initial state is always validated
- **`emitValidated()`:** Use for external data (APIs, forms, storage, WebSocket)
- **`emit()`:** Use for internal transitions (computed values, counters, toggles)
- **`patchValidated()`:** Use for partial external updates
- **Failed validation throws:** State remains unchanged on `BlocValidationError`
- **Helper methods:** `validate()`, `isValid()`, `parse()`, `safeParse()`
- **Performance:** Only validate when crossing boundaries, not on every state change
