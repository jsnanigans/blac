## Introduction

Now that you've seen a basic Cubit, let's make it interactive! In this guide, you'll build a counter that responds to user actions—the "Hello World" of state management.

You'll learn how to:
- Update state in response to user interactions
- Use both `patch()` and `emit()` for state updates
- Call Cubit methods from React components

## The Counter State

First, define your state shape with TypeScript:

```typescript
interface CounterState {
  count: number;
}
```

Using an interface gives you type safety and makes it easy to expand your state later.

## The Counter Cubit

Here's a Cubit with three methods for manipulating the counter:

```typescript
import { Cubit } from '@blac/core';

class CounterCubit extends Cubit<CounterState> {
  constructor() {
    super({ count: 0 }); // Initial state
  }

  increment = () => {
    this.patch({ count: this.state.count + 1 });
  };

  decrement = () => {
    this.patch({ count: this.state.count - 1 });
  };

  reset = () => {
    this.emit({ count: 0 });
  };
}
```

<GuideArticleNote title="Critical: Arrow Functions Required" className="border-red-200 bg-red-50 dark:border-red-900 dark:bg-red-950">
All methods in Cubits MUST use arrow function syntax (`methodName = () => {}`). Regular methods will break in React because `this` binding is lost when passed as callbacks.

```typescript
// ✅ Correct - Arrow function preserves 'this'
increment = () => {
  this.patch({ count: this.state.count + 1 });
};

// ❌ Wrong - Regular method loses 'this' in React
increment() {
  this.patch({ count: this.state.count + 1 }); // Error: 'this' is undefined!
}
```
</GuideArticleNote>

### Understanding State Updates

<GuideArticleNote title="patch() vs emit()">
BlaC provides two methods for updating state:

- **`patch()`** - Performs a shallow merge with existing state. Ideal for updating specific fields without touching others.
- **`emit()`** - Completely replaces the entire state object. Use when resetting or when you need a fresh state.

In our counter, we use `patch()` for increments/decrements (preserving structure) and `emit()` for reset (fresh state).
</GuideArticleNote>

## Using in React

Connect the Cubit to your component and call its methods:

```tsx
import { useBloc } from '@blac/react';

function Counter() {
  const [state, cubit] = useBloc(CounterCubit);

  return (
    <div>
      <h1>{state.count}</h1>
      <button onClick={cubit.increment}>+</button>
      <button onClick={cubit.decrement}>-</button>
      <button onClick={cubit.reset}>Reset</button>
    </div>
  );
}
```

The `useBloc` hook returns:
- `state` - The current state object
- `cubit` - The Cubit instance with all your methods

## How State Updates Work

When you click a button:

1. The button's `onClick` calls a Cubit method
2. The method calls `patch()` or `emit()` with new state
3. BlaC notifies all subscribed components
4. React re-renders components with the new state
5. Your UI updates instantly

All of this happens automatically—you just call the method!

## Type Safety & Benefits

TypeScript provides multiple layers of safety and developer experience improvements:

### Catch Errors Before Runtime

```typescript
// ✅ Correct - TypeScript knows state.count exists
<h1>{state.count}</h1>

// ❌ Error - TypeScript catches the typo immediately
<h1>{state.cnt}</h1>  // Property 'cnt' does not exist on type 'CounterState'

// ✅ Correct - patch accepts partial state
cubit.patch({ count: 5 });

// ❌ Error - patch requires valid keys from CounterState
cubit.patch({ counter: 5 });  // Object literal may only specify known properties
```

### IDE Autocomplete & IntelliSense

As you type `state.`, your IDE instantly shows all available properties:
- `state.count` ✅ (autocompletes)
- Shows type information: `count: number`
- Inline documentation if you add JSDoc comments

This makes development faster and reduces the need to constantly reference documentation.

### Safe Refactoring

Want to rename `count` to `value`? TypeScript makes it safe:

1. Update your interface: `count` → `value`
2. TypeScript immediately highlights **every** location that needs updating
3. Use IDE's "Rename Symbol" to update all references automatically
4. No runtime surprises—if it compiles, your refactor is complete

### Type Inference

TypeScript infers types automatically, reducing verbosity:

```typescript
const [state, cubit] = useBloc(CounterBloc);
// TypeScript knows:
//   state: CounterState
//   cubit: CounterBloc
// No manual type annotations needed!
```

<GuideArticleNote title="Why TypeScript Matters">
In a real application with dozens of Cubits and complex state shapes, TypeScript prevents entire categories of bugs. A typo that would cause a runtime crash in JavaScript becomes a compile-time error you fix in seconds.
</GuideArticleNote>

## Shared State by Default

By default, all components using the same Cubit class share one instance. Try adding multiple `<Counter />` components—they'll all display the same count!

```tsx
function App() {
  return (
    <div>
      <Counter />  {/* All three components */}
      <Counter />  {/* share the same */}
      <Counter />  {/* counter state! */}
    </div>
  );
}
```

## Key Takeaways

- **Arrow functions are required** for Cubit methods
- **patch()** for partial updates, **emit()** for complete replacement
- **useBloc returns [state, instance]** - use both!
- **Call methods directly** from event handlers
- **State is shared** across components by default
